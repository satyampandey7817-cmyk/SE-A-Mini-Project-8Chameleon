### `lib/main.dart` — App Entry Point

**Purpose:** The starting point of the entire Flutter app.

**What it does:**
1. Calls `WidgetsFlutterBinding.ensureInitialized()` to prepare Flutter engine
2. Initializes `NotificationService` singleton
3. Wraps the app in Riverpod's `ProviderScope` for state management
4. Defines `SmartHealthApp` — a `MaterialApp` with the dark glassmorphic theme
5. Contains `AppEntry` widget that controls the app flow:
   - Shows `SplashScreen` for 3 seconds
   - Checks if onboarding is complete → shows `OnboardingScreen` or main app
6. Defines `AppShell` — the main navigation scaffold with a **glassmorphic bottom navigation bar**:
   - 5 tabs: Home, Medicines, Diagnose (Symptom Checker), Appointments, Profile
   - Bottom nav uses `BackdropFilter` for glass effect with `extendBody: true`
   - `IndexedStack` for preserving tab state

**Key classes:** `SmartHealthApp`, `AppEntry`, `AppShell`

### `lib/models/` — Data Models

#### `models.dart`

**Purpose:** Defines ALL data classes (domain entities) used across the app.

**Classes defined:**

| Class | Lines | Description |

| `UserProfile` | ~60 lines | User's personal and health information. Fields: `id`, `name`, `age`, `gender`, `phone`, `email`, `bloodGroup`, `emergencyContactName`, `emergencyContactPhone`, `healthConditions` (List), `allergies` (List), `height`, `weight`, `onboardingComplete`. Has `copyWith()` method, computed `bmi` (weight / height² converted from cm), and `bmiCategory` (Underweight / Normal / Overweight / Obese). |
| `Medicine` | ~40 lines | Medicine with dose tracking. Fields: `id`, `name`, `dosage`, `form` (Tablet/Capsule/Syrup/Injection), `reminderTimes` (List of time strings), `frequency`, `withFood`, `notes`, `isCompleted`, `isReminderOn`, `takenTimes` (List of taken time strings). Computed: `takenCount`, `totalDoses`. Has `copyWith()`. |
| `Appointment` | ~30 lines | Doctor appointments. Fields: `id`, `doctorName`, `specialty`, `dateTime`, `location`, `status`, `notes`. Computed: `isUpcoming` (dateTime after now), `isPast`. Has `copyWith()`. |

| `WaterIntake` | ~10 lines | Daily water tracking. Fields: `date`, `glassCount`, `goal`. Computed: `percentage` (0.0-1.0). Has `copyWith()`. |
| `VitalRecord` | ~10 lines | Health measurement. Fields: `id`, `type` (heartRate/bloodPressure/bloodSugar/weight), `value`, `value2` (for BP diastolic), `recordedAt`, `unit`. |

### `lib/providers/` — State Management

#### `providers.dart`

**Purpose:** Defines ALL Riverpod providers and StateNotifiers that manage application state.

**Repository Providers (singletons):**

| Provider | Type | Returns |
|---|---|---|
| `medicinesRepoProvider` | `Provider<MedicinesRepository>` | Single instance of medicines repo |
| `appointmentsRepoProvider` | `Provider<AppointmentsRepository>` | Single instance of appointments repo |
| `profileRepoProvider` | `Provider<ProfileRepository>` | Single instance of profile repo |
| `checkInRepoProvider` | `Provider<CheckInRepository>` | Single instance of check-in repo |

**State Notifier Providers:**

| Provider | Notifier Class | State Type | Key Methods |
|---|---|---|---|
| `medicinesProvider` | `MedicinesNotifier` | `List<Medicine>` | `addMedicine()`, `updateMedicine()`, `deleteMedicine()`, `markDoseTaken()`, `markAllDosesTaken()`, `toggleReminder()`, `adherenceStreak` (getter) |
| `appointmentsProvider` | `AppointmentsNotifier` | `List<Appointment>` | `addAppointment()`, `updateAppointment()`, `deleteAppointment()`, `upcomingAppointments` (getter), `pastAppointments` (getter), `nextUpcoming` (getter) |
| `profileProvider` | `ProfileNotifier` | `UserProfile` | `updateProfile()`, `completeOnboarding()` |
| `checkInProvider` | `CheckInNotifier` | `DailyCheckIn?` | `saveMood()` |

**Special behavior:**
- `MedicinesNotifier` automatically schedules/cancels local notifications when medicines are added/removed or reminders are toggled
- `MedicinesNotifier.adherenceStreak` counts consecutive days of 100% dose completion

### `lib/services/` — External Service Integration

#### `notification_service.dart`

**Purpose:** Manages local push notifications for medicine reminders.

**Pattern:** Singleton (factory constructor returns same instance)

**Dependencies:** `flutter_local_notifications`, `timezone`

**Initialization (`init()`):**
1. Initializes timezone data
2. Sets up Android notification settings (uses `@mipmap/ic_launcher` as icon)
3. Sets up iOS notification settings (sound, badge, alert permissions)
4. Creates notification plugin instance

**Key methods:**

| Method | What it does |
|---|---|
| `init()` | Initialize notification channels and timezone |
| `requestPermissions()` | Request OS-level notification permissions (iOS/Android 13+) |
| `scheduleMedicineNotification(medicine, userName, timeString)` | Schedules a daily repeating notification at the specified time with medicine name and dose info |
| `cancelMedicineNotifications(medicine)` | Cancels all scheduled notifications for a specific medicine |

### `lib/widgets/` — Reusable UI Components

#### `glass_card.dart` — Glassmorphic Card

**Purpose:** The most-used widget in the app. Wraps any content in a frosted-glass container.

**How it works:**
1. `ClipRRect` clips content to rounded corners
2. `BackdropFilter` applies `ImageFilter.blur(sigmaX: 18, sigmaY: 18)` — blurs everything behind the card
3. Inner `Container` has `glassWhite` fill (8% white) and `glassBorder` (19% white) border
4. Child content rendered on top

**Configurable props:** `child`, `margin`, `padding`, `borderRadius` (default 18), `borderColor`

#### `nebula_background.dart` — Animated Background

**Purpose:** Creates an animated deep-space background with floating gradient orbs.

**How it works:**
1. `SingleTickerProviderStateMixin` drives a 12-second looping `AnimationController`
2. `AnimatedBuilder` rebuilds on every frame
3. 3 `Positioned` circles drift using trigonometric functions:
   - Orb 1 (Electric Blue, 200px): `sin(t * 2π)` × 30 horizontal, `cos(t * 2π)` × 20 vertical
   - Orb 2 (Radiant Pink, 160px): `cos(t * 2π * 1.3)` × 25, `sin(t * 2π * 1.3)` × 35
   - Orb 3 (Neon Green, 140px): `sin(t * 2π * 0.7)` × 20, `cos(t * 2π * 0.7)` × 30
4. Each orb is a `Container` with `RadialGradient` and `BoxDecoration` with `shape: BoxShape.circle`
5. Base layer: `scaffoldGradient` (bgPrimary → bgSecondary)
6. Child content rendered on top via `Stack`

#### `gradient_button.dart` — CTA Button

**Purpose:** Primary call-to-action button used for Save, Confirm, and Add actions.

**Visual:**
- Full-width rounded container with `accentGradient` (blue→pink)
- Optional leading icon
- Label text in white bold
- Glow shadow effect on tap (electricBlue with spread)

**Props:** `label` (String), `onTap` (VoidCallback), `icon` (optional IconData)

#### `accent_bar.dart` — Vertical Color Bar

**Purpose:** Thin vertical colored bar placed on the left edge of list item cards for visual categorization.

**Visual:**
- 4px wide, 44px tall (configurable)
- Rounded corners
- Matching color glow shadow

**Props:** `color`, `height`, `width`
